- `src-tauri/src/operation_log.rs`：追加式结构化操作记录；只保存 action/phase/status、操作 ID、时间、备份目录和计数，不提供自由文本凭据字段。
- `src-tauri/src/process_control.rs`：Windows `tasklist` / `taskkill` 封装；逐 PID 检查命令结果并再次枚举，只有确认无存活 Codex 进程才成功；非 Windows明确返回 unsupported。
- `src-tauri/src/relay_verify.rs`：使用 rustls 客户端请求 `<base>/models`；Bearer 认证、10 秒超时、禁止重定向、4 MiB 响应上限，只返回状态/模型数量并脱敏错误。
- `src-tauri/src/runtime_store.rs`：固定 `plus` / `relay` 两槽位；损坏 metadata fail-closed，DPAPI 加密认证，覆盖前归档，三文件写失败时恢复并验证旧快照；relay 仅允许 HTTPS 或 loopback HTTP，空 Key 保留旧凭据。
- `src-tauri/src/runtime_switcher.rs`：执行 `preflight -> backup -> current→shared sync -> atomic auth/config overlay -> shared→current sync -> exact verify -> receipt`；仅 exact 同态 no-op，切换中再次检查进程；任一步失败恢复 current/shared 两份已验证快照。
- `src-tauri/src/session_manager.rs`：合并 current/shared 会话视图；所有硬删除都要求确认，在两根创建完整快照后用 Immediate 事务删除 thread/边表并原子更新文件，跨根部分失败恢复两份快照；恢复可见只写当前 Home。
- `src-tauri/src/session_sync.rs`：JSONL-first 双向同步；2 秒 SQLite busy timeout、事务与 `quick_check`；未知必需 schema 失败而非忽略。热同步不重写已有 live JSONL；关闭态才流式原子更新 provider 或用增长文件替换旧前缀。分叉文件按内容哈希保留，`session_index.jsonl` 只追加允许的未归档会话。
- `src-tauri/src/skill_manager.rs`：两个固定 Skill 的编译期文件 allowlist、manifest/SHA-256 状态检测、绝对路径/reparse 防护、同卷 stage/旧目录备份/原子激活、带 phase 的崩溃 journal 恢复与 DPAPI 配置保存；备份尚未创建时恢复不得删除用户旧目录。
- `src-tauri/src/update_check.rs`：固定请求本仓库最新正式 GitHub Release；限制响应大小、禁止重定向、校验 stable SemVer、比较当前版本、截断更新说明，并提供受控资产元数据给安装模块。
