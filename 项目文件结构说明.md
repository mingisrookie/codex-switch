- `src-tauri/src/codex_paths.rs`：统一解析 Codex home、`sqlite_home`、`state_5.sqlite`、`sessions/` 和 `session_index.jsonl`；用户真实路径会读取 `config.toml sqlite_home` / `CODEX_SQLITE_HOME`，共享池路径固定使用自身目录。
- `src-tauri/src/codex_home.rs`：Codex home 文件状态扫描、解析后 SQLite home 展示与 `auth.json` 结构摘要。
- `src-tauri/src/redaction.rs`：敏感字段识别与 JSON 脱敏。
- `src-tauri/src/config_patch.rs`：保护式 `config.toml` patch 计划。
- `src-tauri/src/session_scan.rs`：按 `codex_paths` 解析出的 `state_5.sqlite` 与 `sessions/**/*.jsonl` 做只读会话扫描、dry-run 去重，并读取 title / preview / provider / archived 等展示字段。
- `src-tauri/src/backup.rs`：切换/同步前快照备份和 SHA-256 manifest；备份解析后的 `state_5.sqlite`、WAL/SHM、`auth.json`、`config.toml`、`session_index.jsonl` 和会话正文清单。
- `src-tauri/src/crypto.rs`：Windows DPAPI 加密/解密封装，用于运行态和 legacy profile 的 `auth.enc`。
- `src-tauri/src/profile_store.rs`：旧账号/API profile 元数据、加密 `auth.enc`、provider config overlay；保留后端兼容，当前主 UI 不再使用。
- `src-tauri/src/process_control.rs`：Windows `tasklist` / `taskkill` 封装，检测并关闭 Codex 进程。
- `src-tauri/src/runtime_store.rs`：Codex 账号 / 单个 API 中转站运行态存储，使用 DPAPI 加密 `auth.json`；中转站生成 `openai_custom` provider 配置模板，未显式 scheme 时默认补 `https://`，不在 config 中写明文 Key 或 env 引用。
- `src-tauri/src/runtime_switcher.rs`：运行态切换执行器，负责按真实 `sqlite_home` 与共享会话池同步、备份、原子替换目标运行态的 `auth.json` / `config.toml`、切换后写回会话；切换后按目标运行态 provider 归一化当前 Codex home 的会话索引与 JSONL 元数据。
- `src-tauri/src/switcher.rs`：旧 profile 切换执行器，负责备份、替换 `auth.json`、patch `config.toml`；保留后端兼容。
- `src-tauri/src/session_manager.rs`：会话管理专用后端模块，合并当前 Codex Home 与 `%APPDATA%\codex-switch\shared-sessions`，当前 Home 优先、shared-sessions 补缺；负责删除所选、恢复可见和写入前备份。
- `src-tauri/src/session_sync.rs`：JSONL-first 会话写入同步，事务插入 / 修复 `threads` 与相关边表、复制 JSONL 并合并 `session_index.jsonl`；只同步存在正文 JSONL 且未归档的会话，跳过 DB-only 孤儿 thread 和已归档 thread；重复 thread 会继续修复缺失 `rollout_path`，provider 感知同步只更新 `session_meta.payload.model_provider` 元数据，不改用户/助手正文。

- `vite.config.ts`：Vite/Vitest/React 配置。
